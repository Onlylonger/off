## 为什么你的开发需要多版本Node.js管理

想象一下这样的场景：你正在开发一个重要的企业项目，使用的是最新的Node.js 20版本。突然，客户报告了一个在旧版系统上运行的遗留应用出现了问题——这个应用只兼容Node.js 14。传统上，你可能需要：

1. 卸载当前版本的Node.js

2. 安装旧版本

3. 测试修复问题

4. 再切换回新版本继续开发

这个过程不仅耗时，还容易出错。而有了多版本管理工具，你只需一条命令就能在版本间无缝切换：

```bash
nvm use 14.18.1 # 切换到遗留项目需要的版本
# 修复问题...
nvm use 20.13.0 # 切换回主项目
```

## 主流Node.js版本管理工具对比

### nvm (Node Version Manager) - 开发者的首选

- 适用平台：macOS/Linux

- 特点：
  - 轻量级，纯bash脚本实现

  - 支持所有Node.js版本

  - 简单易用的命令

安装只需一行命令：

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
```

常用命令示例：

```bash
nvm install 18 # 安装Node.js 18的最新版本
nvm install 20.13.0 # 安装特定版本
nvm use 18 # 切换到18版本
nvm alias default 20 # 设置默认版本
```

### n - 简洁高效的工具

- 适用平台：全平台（通过npm安装）

- 特点：
  - 极简设计，命令更简单

  - 通过npm全局安装

  - 自动处理权限问题

安装与使用：

```bash
npm install -g n # 全局安装
n 20.13.0 # 安装并使用20.13.0版本
n # 查看已安装版本并切换
```

### fnm (Fast Node Manager) - 速度之王

- 适用平台：全平台

- 特点：
  - 使用Rust编写，速度极快

  - 支持.nvmrc和.node-version文件

  - 自动版本切换

安装与使用：

```bash
# 使用Homebrew安装(macOS)
brew install fnm

# 初始化shell配置
eval "$(fnm env --use-on-cd)"

fnm install 20 # 安装Node.js 20
fnm use 20 # 使用20版本
```
